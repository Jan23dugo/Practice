Player-Related Tables

Players

Stores information about user accounts that can have multiple character profiles.

Column Data Type Constraints Default Description
PlayerID INT PK, AUTO_INCREMENT - Unique identifier for each player account
Username VARCHAR(50) NOT NULL, UNIQUE - Player's unique username used for login
CreatedAt DATETIME - CURRENT_TIMESTAMP Timestamp when the account was created
LastLoginAt DATETIME - CURRENT_TIMESTAMP Timestamp of the player's most recent login
Relationships:
  • Players → Profiles (One-to-Many): One player account can have multiple character profiles
  • Players → GameSettings (One-to-One): Each player has one set of game settings

Profiles

Represents individual character profiles that belong to player accounts.

Column Data Type Constraints Default Description
ProfileID INT PK, AUTO_INCREMENT - Unique identifier for each character profile
PlayerID INT FK - References the player account that owns this profile
ProfileName VARCHAR(50) NOT NULL - Name of the character profile
CreatedAt DATETIME - CURRENT_TIMESTAMP Timestamp when the profile was created
LastPlayedAt DATETIME - CURRENT_TIMESTAMP Timestamp when the profile was last played
TotalPlayTime INT - 0 Total time (in seconds) spent playing with this profile
CurrentDungeonID INT FK, NULL NULL References the dungeon the player is currently in, if any
CurrentFloor INT - 1 The current floor number in the dungeon
Relationships:
  • Profiles → Players (Many-to-One): Each profile belongs to one player account
  • Profiles → PlayerStats (One-to-One): Each profile has one set of player stats
  • Profiles → PlayerWeapons (One-to-Many): A profile can own multiple weapons
  • Profiles → PlayerInventory (One-to-Many): A profile can have multiple inventory items
  • Profiles → Dungeons (Many-to-One): A profile can be in one dungeon at a time
  • Profiles → SavedProgress (One-to-Many): Each profile can have multiple saved game states

PlayerStats

Stores the current stats for each player profile.

Column Data Type Constraints Default Description
StatID INT PK, AUTO_INCREMENT - Unique identifier for each stat record
ProfileID INT FK - References the profile these stats belong to
Health INT - 100 Current health points
MaxHealth INT - 100 Maximum possible health points
Mana INT - 100 Current mana points for using skills
MaxMana INT - 100 Maximum possible mana points
MovementSpeed FLOAT - 1.0 Player movement speed multiplier
Armor INT - 0 Amount of damage reduction from armor
Currency INT - 0 Amount of in-game currency owned
Relationships:
  • PlayerStats → Profiles (One-to-One): Each stat record belongs to exactly one profile

GameSettings

Stores game settings preferences for each player account.

Column Data Type Constraints Default Description
SettingID INT PK, AUTO_INCREMENT - Unique identifier for each settings record
PlayerID INT FK - References the player account these settings belong to
ControlType ENUM - 'Touch' Preferred control scheme (Touch, Joystick)
SoundVolume FLOAT - 1.0 Sound effects volume (0.0-1.0)
MusicVolume FLOAT - 1.0 Background music volume (0.0-1.0)
LastUpdated DATETIME - CURRENT_TIMESTAMP Timestamp when settings were last updated
Enum Values:
  • ControlType: 'Touch', 'Joystick'
Relationships:
  • GameSettings → Players (One-to-One): Each settings record belongs to exactly one player account
